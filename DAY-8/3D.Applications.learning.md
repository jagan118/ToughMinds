## **PHASE 1: FOUNDATIONS (Day 1)**

### What You Need to Learn

**1. WebGL vs Three.js**
- WebGL = low-level graphics API (complex math)
- Three.js = easier library that handles WebGL for you
- **You'll use:** Three.js (way simpler)

**2. 3D Coordinate System**
- 2D: x, y (flat)
- 3D: x, y, z (depth added)
- Everything in 3D has position (x, y, z)

**3. The 3 Essential Components**
- **Scene:** Container for all 3D objects (like a stage)
- **Camera:** Where you're viewing from (your eye position)
- **Renderer:** Draws everything to screen

**4. Basic Shapes (Geometries)**
- Cube, Sphere, Cylinder, Plane
- These are your building blocks for equipment

**5. Materials (How Things Look)**
- Color, shininess, reflection
- Different materials look different under light

**6. Lights (Illumination)**
- Ambient light (background glow)
- Directional light (like sun)
- Point light (like light bulb)

**7. The Rendering Loop**
- Traditional web: Event → Response
- 3D apps: Loop 60 times/second, redraw everything
- This is the heartbeat of 3D apps

## **PHASE 2: MAKING THINGS INTERACTIVE (Day 2-3)**

### What You Need to Learn

**8. Data → Visual Changes**
- Real data (temperature, pressure) → Color/position/size changes
- RxJS + Three.js = real-time updates

**9. Raycasting (Click Detection)**
- User clicks on screen → Detect which 3D object they clicked
- Like casting a line from camera through mouse

**10. Grouping Objects**
- Multiple parts = group them
- Move parent = all children move together
- Like organizing a pump (motor + pipes + sensor)

**11. Smooth Animations**
- Don't jump from A to B instantly
- Animate over time (color fade, smooth movement)

---

## **PHASE 3: REAL-TIME DATA (Day 4-5)**

### What You Need to Learn

**12. WebSocket + 3D**
- Backend sends data via Socket.io
- Frontend receives → Updates 3D objects immediately
- Multiple users see live changes

**13. State Management**
- Keep data in service/store (separate from 3D objects)
- 3D objects = just visual representation
- When data changes → update store AND mesh

**14. Performance Optimization**
- 1000 objects = FPS drops
- Solution: Simplify far-away objects, hide objects outside view
- Keep 60 FPS smooth

## **KEY CONCEPTS EXPLAINED SIMPLY**

### **Rendering Loop (Most Important)**
```
while (true) {
  1. Update → Get new sensor data, change positions/colors
  2. Render → Draw everything to screen
  3. Repeat 60 times/second
}
```

### **Data Flow for SCADA**
```
Backend Sensors
    ↓
Socket.io (real-time)
    ↓
Service (stores data)
    ↓
Update 3D objects (mesh properties)
    ↓
Renderer draws → User sees changes
```

### **3D Space Example**
```
Factory Floor Visualization:
- Pump at (2, 1, 5) = 2m right, 1m up, 5m forward
- Motor at (0, 1, 5) = center, 1m up, 5m forward
- Tank at (-3, 2, 10) = 3m left, 2m up, 10m forward

Color based on status:
- Green = Running ✓
- Red = Error ✗
- Yellow = Warning ⚠
```

## **MENTAL MODEL: 3D VS 2D WEB APPS**

| Aspect | 2D Web App | 3D Web App |
|--------|-----------|-----------|
| Rendering | DOM elements | 3D scene every frame |
| Updates | Event-driven | Continuous loop (60 FPS) |
| Interaction | Click buttons/links | Click 3D objects in space |
| Position | CSS (pixels) | 3D coordinates (x, y, z) |
| Performance | Add more elements | FPS drops with complexity |
| Learning | HTML/CSS/JS | Math + graphics concepts |
