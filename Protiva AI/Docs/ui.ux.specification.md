# Philosophy

## Our Golden Rule

> **The user should never feel like they're building a website.**

Instead, they should feel like they're introducing themselves.

That's the biggest difference between Portiva AI and platforms like Wix, WordPress, Squarespace, or other AI website builders.

Instead of asking:

❌ Choose Template

❌ Pick Colors

❌ Create Pages

❌ Configure Layout

We ask:

# UX Principles

Every interaction should follow these rules:

1. Minimize typing with AI, uploads, and smart defaults.
2. Show progress so users know where they are.
3. Allow manual control without overwhelming beginners.
4. Never lose work—autosave drafts continuously.
5. Give clear, actionable feedback instead of generic errors.
6. Keep the portfolio preview as close as possible to the final published experience.

# Frontend Architecture Notes

Build the UI using a **component-first** approach.

Examples:

```
components/
  button/
  input/
  card/
  modal/
  ai-chat/
  portfolio-section/
  theme-selector/
  media-uploader/
  score-card/
```

Pages should compose these reusable components rather than duplicating UI logic.
